Github教程总结

github命令教程

git是一个分布式版本系统，它的最大好处之一是在本地工作完全不需要考虑远程库的存在，也就是有没有联网都可以正常工作，而SVN在没有联网的时候是拒绝干活的
熟悉三个

工作区-add-版本库（暂存区(stage/index)-commit-当前分支(当前分支默认master)）

$ git add readme.txt 把文件添加到仓库 (-A全部)
$ cat readme.txt 读取readme.txt
$ git commit -m "wrote a readme file"把文件提交到仓库

$ git status 时刻掌握仓库当前的状态
$ git diff readme.txt查看difference

撤销版本库（hard连带撤销本地）

$ git log 从最近到最远的提交日志, 穿梭历史
$ git reflog 用来记录你的每一次命令 要重返未来
$ git log --pretty=oneline
$ git reset --hard HEAD^ HEAD^^上上个版本 HEAD~100上100个版本
$ git reset --hard 3628164（前面几位commit id）
git reset –soft：
回退到某个版本，只回退了commit的信息，不会恢复到index file一级。如果还要提交，直接commit即可　
git reset –hard：
彻底回退到某个版本，本地的源码也会变为上一个版本的内容　　

$ git diff HEAD -- readme.txt 命令可以查看工作区和版本库里面最新版本的区别
每次修改，如果不add到暂存区，那就不会加入到commit中。

撤销工作区

$ git checkout -- readme.txt （还没add）意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：

  • 一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；

  • 一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态

改乱了工作区某个文件的内容，想直接丢弃工作区的修改时，用命令
$ git checkout -- file

撤销缓存区

$ git reset HEAD file（add后）可以把暂存区的修改撤销掉（unstage），重新放回工作区HEAD时，表示最新的版本。后面再$ git checkout -- readme.txt

删除版本库文件

$ git rm test.txt + $ git commit -m "remove test.txt"版本库删除丢失最近一次提交后你修改的内容。

添加到远程库

可以把分支最新修改直接推送给远程仓库

-u参数直接推送并关联远程仓库
(关联远程仓库+推送到远程库上)
克隆
$ git clone xxx

分支管理

创建分支
$ git checkout -b dev -b参数表示创建并切换=
$ git branch dev+
$ git checkout dev
$ git branch 命令查看当前分支
$ git merge dev 将dev分支合并到当前分支 （fast forward模式合并就看不出来曾经做过合并。
当出现合并冲突，需要手动解决冲突
$ git branch -d dev删除dev分支
$ git log --graph --pretty=oneline --abbrev-commit
比较好的查看分支记录

普通模式合并，合并后的历史有分支
$ git merge --no-ff -m "merge with no-ff" dev 合并 创建一个新的commit

修复bug时，我们会通过创建新的bug分支进行修复
手头工作没有完成时，先把工作现场$ git stash一下，然后去修复bug，修复后，再`$ git stash pop，回到工作现场。 $ git stash list查看

开发一个新feature，最好新建一个分支；

如果要丢弃一个没有被合并过的分支，可以通过$ git branch -D <name>强行删除。

查看远程库信息，使用$ git remote -v；

推送分支

本地新建的分支如果不推送到远程，对其他人就是不可见的；

从本地推送分支，使用$ git push origin branch-name，如果推送失败，先用git pull抓取远程的新提交；

分支协作

在本地创建和远程分支对应的分支，使用$ git checkout -b branch-name origin/branch-name，本地和远程分支的名称最好一致；

建立本地分支和远程分支的关联，使用$ git branch --set-upstream branch-name origin/branch-name；

从远程抓取分支，使用$ git pull，如果有冲突，要先处理冲突。

$ git tag -a <tagname> -m "blablabla..."可以指定标签信息；

标签管理

用git show 查看标签信息：
$ git tag可以查看所有标签。
$ git tag -d v0.1删除标签
$ git push origin v1.0推送某个标签到远程
$ git push origin --tags一次性推送全部尚未推送到远程的本地标签
删除远程书签

  • $ git tag -d v0.9
  • $ git push origin :refs/tags/

配置忽略文件

.gitignore以斜杠“/”开头表示目录；

　　以星号“*”通配多个字符；

　　以问号“?”通配单个字符

　　以方括号“[]”包含单个字符的匹配列表；

　　以叹号“!”表示不忽略(跟踪)匹配到的文件或目录；

游戏之我见

游戏之我见

＂玩家想玩什么”不能给游戏设计者提供设计思路，玩家只是提供一个模糊的概念，但这个概念无法提供一个确切的需求。所以一个策划设计一个模糊概念下的 “好玩的” 游戏并不能跟着玩家走。但玩家能提供一个大概的方向，比如手游下，休闲化、片碎化游戏玩家群众更大。
读《游戏改变世界》能大概给我提供一个玩家心理学的角度的分析方法。比如对游戏忌惮害怕，怀疑自己错失了真正的生活（玩家之悔）。游戏结束，现实开始时迷失方向虚拟体验的财富。有什么解决方法呢？
权宜之计解决：

  1. 提升游戏的一个思考高度，也就是能让你思考的游戏。
  2. 还有就是休息奖金，通过奖励鼓励你休息，保证现实生活的轨道。鼓励老玩家的回归。

总结起来游戏有四个特征 :目标 规则 反馈 自愿参加。

  • 比如高尔夫，玩家目标就是将球打进洞。
  • 规则就是你不能用手将球拿进洞，这违反了游戏的乐趣。

  • 而反馈的一个就是奖励，有两个角度：自成目的，内在奖励的活动产生持久的幸福。
    外在奖励对幸福没有帮助，容易形成享乐适应。反馈有多种，可以是图形，可以是数字，甚至音乐。反馈力度大，玩家在极限工作条件下投入状态最高，就会进入心流状态无法自拔—-俄罗斯方块。第一次接触心流的概念是从陈星汉游戏中获得，云，花，风之旅人都是意在情感表达，玩家能在游戏获得一些不一样的情感—-心流。

    心流 (flow) 定义为一种将个人精神力完全投注在某种活动上的感觉；

    心流产生时同时会有高度的兴奋及充实感。并非受挫而获得，而是挫折出现时移向另外一个类型，通过磨练两种以上类型的技能（而不是一种）

    玩家将无需通过漫长的受挫时期去到达精通/满足门槛。但太多升级道具或硬核技能将阻碍部分玩家。
    复合流更多出现在一些。除了心流，我在gameboom上还看到一个概念：复合流。更多出现在类似街霸中的心理博弈。

  • 自愿参与就不用太多解释了吧( ´◔ ‸◔’)，比如我讨厌FPS，让我玩的话听到枪声我会疯的。游戏应该是创造性成就和能力带给人的愉快和满足感。
    游戏拥有更宏大的意义。比如文章开头，3000年前吕底亚人发明骰子
    抓子等游戏展开集体游戏逃避饥荒。游戏让饥饿的生活变得可以承受。现在有个游戏化的洞察的研究，游戏能重塑现实。用游戏设计所知的一切弥补现实中的不足，比如教育游戏。所以对游戏不了解及从未进入游戏这个圈子应不能对游戏有过多的偏见。衷心祝愿游戏能作为一种艺术在未来越来越好。

现实已经破碎，我们需要创建游戏修复它。

书单计划

github第一篇列个书单计划

这个月

《任天堂传奇》
《编程之美》
《算法（第四版）》
《游戏引擎架构》
《游戏之旅-我的编程感悟》
《大话设计模式》
《龙枪编年史》
《诸神的星空》

kindle看完《doom启示录》毫无疑问，确实是一本好书，据说 很多人因为这本书热血沸腾投身游戏行业，两个约翰从最初合作无间到最后分道扬镳确实让人感慨。两个人，一个投身技术。一个醉心设计。
约翰卡马克是一位孤傲的程序员，对他来说要学的东西很多，对此他不断追求。游戏引擎、火箭、虚拟现实。如他的格言：

“如果你想动手开发什么全新的技术，你不需要几百万美元的资金，你只需要在冰箱里放满比萨和可乐，再有一台便宜的计算机，和为之献身的决心。”

对技术的不断追求，让人不由得尊敬，而罗梅洛则是将卡马克引擎用得淋漓尽致的一个设计师。没有罗就没有doom。游戏对罗梅洛来说是在生活中玩的，他创造死亡竞技场。各地玩家聚集在一起打败这个Icon of Sin。

每个人都有无法实现的梦想。或许是那梦想需要太多时间和金钱，譬如开跑车驰骋，驾飞机翱翔；或许是那梦想太过于离谱，譬如与异形进行星球大战，与吸血鬼拼个刺刀见红；或许是那梦想会违反法律，譬如痛殴老板，夜半尾行。但不管能否实现，它们总盘旋在你脑海里，每一天，让你浮想连翩。这就是为什么会有一个上亿美元的产业来帮助人们实现各种光怪陆离的白日梦；
